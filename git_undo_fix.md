# Git変更取り消し・修正まとめ
## 1. git reset （ステージングの解除やコミットのやり直し）
ステージに挙げた変更を取り消したり、コミットをやり直したりするときに使う。
```bash
git reset ファイル名
```
> ステージからファイルを外す（変更は残る）
```bash
git reset --soft HEAD^
```
> 直前のコミットを取り消してステージに戻す（変更は残る）
```bash
git reset --hard HEAD^
```
> 直前のコミットを完全に取り消す（変更も破棄）

## 2. git checkout (ファイル変更取り消し)
作業中のファイルの変更を取り消して、最後にコミットした状態に戻す
```bash
git checkout -- ファイル名
```

## 3. commit --amend(直前のコミット修正)
直前にコミットしたメッセージを修正したり、ファイルの追加をやり直したりするときに使う。
```bash
git commit --amend
```
>直前のコミットをやり直す（エディタが開きメッセージ編集可能）
- ファイル変更と一緒にやる場合
1. 変更をステージに追加
```bash
git add ファイル名
```
2. amend実行
```bash
git commit --amend
```
既にリモートにプッシュ済みのコミットをamendで修正すると履歴が変わるので注意。
